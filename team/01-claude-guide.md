# Инструкция 1. Работа с Claude

Эта инструкция для человека, который впервые работает с ИИ-ассистентом Claude. Если ты уже пользуешься Claude — пролистай, возьми что полезно, остальное пропусти.

---

## Что такое Claude и зачем он нам

Claude — это ИИ-ассистент от компании Anthropic. Ты пишешь ему сообщения, он отвечает. Как переписка с очень образованным коллегой, который никогда не устаёт и никогда не обижается.

Для нашей команды Claude решает четыре задачи:

1. **Редактура текстов.** Написал черновик — Claude помогает найти слабые места, предложить альтернативы, почистить от канцелярита и «нейросетевого языка».

2. **Сборка сайтов.** Claude Code for Web — специальная версия Claude, которая умеет писать код сайта прямо в наш GitHub. Ты описываешь словами что нужно, он делает.

3. **Обсуждение структуры.** Прежде чем писать тексты лендинга, нужно понять его структуру. Claude помогает разложить задачу, найти логику, придумать подходы.

4. **Справка и разбор.** Если что-то непонятно в чужом тексте, в данных, в чужом коде — Claude объясняет.

## Два Claude: для разговора и для кода

У нас в работе две разные версии Claude, это важно не путать:

### Обычный Claude — на claude.ai

Это чат-окно, похожее на мессенджер. Ты пишешь сообщения, он отвечает текстом. Здесь происходит:

- Редактура текстов
- Обсуждение структуры лендинга
- Поиск формулировок
- Разборы и справки

Ссылка: **https://claude.ai**

### Claude Code for Web — на claude.ai/code

Тоже чат-окно, но умеет больше. Он подключается к нашему GitHub, читает файлы нашего сайта, вносит в них изменения, создаёт новые файлы, тестирует результат.

Здесь происходит:

- Сборка нового лендинга на основе готовых текстов
- Правки существующих лендингов
- Добавление новых блоков, секций, картинок

Ссылка: **https://claude.ai/code**

**Ключевое различие для тебя:**

- В обычный Claude ты приходишь **с идеей или черновиком**, уходишь **с отредактированным текстом**
- В Claude Code for Web ты приходишь **с готовым текстом и описанием задачи**, уходишь **с рабочим лендингом в preview-версии**

Обычно сначала работаешь в обычном Claude (готовишь тексты), потом в Claude Code for Web (собираешь сайт).

## Первое знакомство с обычным Claude

Зайди на https://claude.ai, залогинься.

Что ты видишь:

- Слева — список твоих прошлых разговоров (вначале пусто)
- Справа — область для нового разговора
- Снизу — поле ввода, куда ты пишешь сообщение

**Первое правило работы с Claude: качество ответа зависит от качества вопроса.**

Плохой вопрос: «напиши текст для лендинга»  
Хороший вопрос: «помоги мне написать заголовок для лендинга о сервисе онлайн-оценки кандидатов для крупных работодателей. Целевая аудитория — HR-директора. Хочу чтобы заголовок был в деловом регистре, короткий, с конкретикой. Вот три моих черновика: [перечисляешь]»

Разница в том, что во втором случае Claude получает контекст:
- Кто ты
- Кто читатель
- Какой регистр
- С чем ты уже работаешь

Чем больше контекста — тем лучше ответ.

## Как построить хороший первый запрос

Запомни формулу:

**Контекст → Задача → Что уже есть → Что ожидаешь на выходе**

Пример плохого запроса:

> Напиши мне три заголовка для лендинга о найме.

Пример хорошего запроса:

> Я работаю в команде Исследовательской лаборатории hrasc-lab.ru. Мы делаем инструменты оценки кандидатов для крупных работодателей.
>
> Мне нужно написать заголовок для нового лендинга — он будет посвящён нашему исследованию рынка труда «Великое сжатие». Исследование про то, как сильно сократилось число вакансий при росте числа резюме, и что это значит для компаний.
>
> Адресат — HR-директора крупных компаний (1000+ сотрудников). Регистр — деловой, консалтинговый, без пафоса и без нейросетевого языка.
>
> Вот пример тона, который мы держим на основном лендинге: «Найм работает иначе. Мы строим инструменты под новую логику рынка труда.»
>
> Предложи 3-5 вариантов заголовка для лендинга исследования. Под каждым — короткое объяснение, что он делает и почему такой.

Второй запрос в 5 раз длиннее, но ответ будет в 10 раз качественнее.

## Три документа, которые тебе пригодятся в работе

Анастасия написала три документа, которые задают тон и правила команды. Их стоит **прикладывать в начало большинства сессий с Claude**, чтобы он сразу понимал контекст.

Где найти: в репозитории `hrasc-lab` в папке `team/` и `docs/`.

### 1. how-to-work-with-me.md

Это документ, написанный для работы с Анастасией. Ты можешь его переиспользовать частично — там описаны:
- Стиль работы (плотность, без пафоса, без канцелярита)
- Принципы редактуры
- Что можно, что нельзя

Прикладывай в чат с Claude, когда работаешь над текстами для наших лендингов — чтобы Claude писал в правильном регистре.

### 2. 03-text-tone.md

Твой рабочий документ по тональности. Более конкретный, чем документ Анастасии, с примерами «хорошо/плохо». В начале сложной сессии по редактуре — прикладывай его.

### 3. design-system.md

Дизайн-система Лаборатории (цвета, шрифты, сетка). Для работы в обычном Claude обычно не нужна — она про визуал. Нужна в Claude Code for Web.

## Как Claude «забывает» между разговорами

Важная особенность: **каждый новый разговор с Claude — это чистый лист**. Claude не помнит, о чём вы говорили в прошлой сессии.

Это значит:

- Не надо начинать «продолжаем с прошлого раза» — он не знает, о чём было
- В начале каждой новой сессии прикладывай нужные документы заново
- Если работа над одной задачей растянута на несколько дней — храни важные результаты в документах, не надейся на «память» Claude

Есть одно исключение: **внутри одной сессии** Claude помнит всё, что было сказано. Пока ты в одном чат-окне — история есть.

---

## Когда начинать новую сессию

Чем длиннее сессия — тем «тяжелее» становится Claude, и тем выше риск, что он начнёт путаться или отвечать медленнее.

Признаки, что пора начинать новую сессию:

- Отвечает значительно медленнее, чем вначале
- Забывает, о чём ты говорил 10 сообщений назад
- Сам пишет что-то вроде «я могу потерять часть контекста»
- Начал повторяться или перестал предлагать новые идеи

**Как правильно переходить в новую сессию:**

1. В текущей сессии напиши: «Сделай краткое резюме того, до чего мы дошли: ключевые решения, утверждённые тексты, открытые вопросы»
2. Получи резюме, сохрани его (скопируй в документ или пришли себе на почту)
3. Открой новую сессию, приложи нужные документы + резюме
4. Продолжай с того места, где остановились

Это занимает 3 минуты, но спасает от потери контекста.

## Как задавать уточняющие вопросы Claude

Иногда Claude выдаёт ответ, который не совсем попадает. Как корректировать:

### Скажи точно, что не подходит

Плохо: «нет, не так»  
Хорошо: «второй вариант слишком рекламный, в нём слово "уникальный". Первый и третий ок, но в третьем странный переход во втором предложении»

Чем конкретнее указание — тем точнее правка.

### Покажи пример того, что хочешь

Если правильный тон тяжело описать словами — покажи пример из другого источника или своих старых текстов. «Хочу вот как здесь: [вставляешь пример]».

### Проси объяснить, а не переделать

Если Claude написал что-то странное, прежде чем переделывать — спроси «почему ты так написал?». Иногда оказывается, что он неправильно понял задачу. Тогда правильнее исправить формулировку задачи, чем править результат.

## Типичные ошибки новичков

### Ошибка 1. Задаёшь вопрос, получаешь ответ, веришь на слово

Claude ошибается. Регулярно. Не злонамеренно, просто так работает. Всегда проверяй факты:

- Статистика — гугли первоисточник
- Юридические моменты — проверяй у юриста, а не у Claude
- Технические штуки — проверяй на preview, а не «он сказал, что должно работать»

### Ошибка 2. Пишешь длинное сообщение одной простыней

Плохо:
> помоги сделать лендинг для исследования у нас было исследование про рынок труда называется великое сжатие цель рассказать HR директорам про падение вакансий и рост резюме вот материалы [длинная простыня] сделай структуру и заголовки

Хорошо:
> Работаем над лендингом для исследования «Великое сжатие». 
>
> **Контекст.** [два-три предложения о том, что за исследование]
>
> **Адресат.** [один-два предложения — кто читатель]
>
> **Материалы исследования.** [прикрепляешь документ или вставляешь ключевые цифры]
>
> **Что нужно на этом шаге.** Предложить структуру лендинга — какие блоки, в каком порядке, о чём каждый.

Структура + заголовки помогают Claude понимать, где что. Плюс тебе самому проще перечитывать.

### Ошибка 3. Начинаешь переписывать все сразу по первой правке

Claude иногда даёт неудачный вариант. Не злись, не переписывай всё с нуля. Скажи «первый абзац оставь, второй перепиши вот в таком направлении». Claude хорошо работает с точечными правками.

### Ошибка 4. Ожидаешь «прочтения мыслей»

Claude не знает твоих внутренних задач, истории команды, политических нюансов. Если у тебя в голове есть контекст «но у нас тут есть конфликт с одним из блоков, его нельзя называть напрямую» — напиши это явно.

### Ошибка 5. Не задаёшь вопрос про качество ответа

Когда получил ответ, всегда спрашивай себя:
- Это реально подходит под мой контекст?
- Я бы это написал сам?
- Тон попал в нужный регистр?

Если ответ «нет» хотя бы на один пункт — проси переделать, не неси дальше.

## Claude Code for Web — особенности

Это отдельный продукт на **https://claude.ai/code**. Логинишься тем же аккаунтом, что в обычном Claude.

Главное отличие: Claude Code for Web **подключен к нашему GitHub**. Он:

- Видит все файлы нашего репозитория
- Может создавать новые файлы
- Может редактировать существующие
- Создаёт ветки (черновые версии кода), которые можно смотреть в preview
- Коммитит и пушит изменения сам — ты ничего не нажимаешь

Тебе не нужно уметь программировать. Ты описываешь словами что нужно сделать — Claude Code делает.

### Как работать с Claude Code for Web

**Базовый принцип:**

1. Готовишь тексты в обычном Claude
2. Сохраняешь тексты в документ (например, `landing-copy-research.md`)
3. Загружаешь документ в репозиторий в папку `docs/`
4. Открываешь Claude Code for Web
5. Создаёшь новую сессию
6. Пишешь промпт: «Сделай новый лендинг на основе `docs/landing-copy-research.md`, используй дизайн-систему `docs/design-system.md`»
7. Ждёшь результат — получаешь ссылку на preview
8. Смотришь preview, даёшь правки, получаешь новый preview
9. Когда готово — отправляешь ссылку Анастасии

### Что важно помнить при работе с Claude Code

- **Сессии длятся долго.** Одна сборка лендинга — это 10-30 минут работы. Не закрывай вкладку
- **Таймауты случаются.** Иногда связь рвётся на середине. Если это случилось — пиши «продолжи с места, где остановился». Claude Code помнит контекст сессии
- **Не все команды сразу срабатывают.** Иногда Claude Code задаёт уточняющие вопросы перед тем, как начать. Отвечай на них — это нормально
- **Preview появляется не сразу.** После того как Claude Code закончил работу, нужно 1-3 минуты, чтобы Vercel собрал preview-версию сайта. Не паникуй, если ссылка сразу не открывается

### Промпт для Claude Code — формат

Claude Code работает лучше всего, когда промпт структурирован. Типовой шаблон:

```
Привет. Задача — [одна фраза].

**Контекст.**
- Работаем в репозитории hrasc-lab
- Для ориентира прочитай: docs/landing-copy-[название].md и docs/design-system.md

**Что нужно сделать.**
1. Создать новую ветку (называй как угодно, на твоё усмотрение)
2. [конкретные шаги, что именно делать]
3. [ещё шаги]

**Чего НЕ делать.**
- Не трогай main
- Не меняй существующие лендинги
- [другие ограничения]

**После работы.**
Пришли ссылку на preview.
```

Примеры промптов — в документе `OPERATIONS.md`.

## Когда просить помощь Анастасии

Не бойся это делать. Но сначала проверь, что ты:

1. Прочитал инструкции в папке `team/`
2. Попробовал сформулировать вопрос сам и попросить Claude помочь разобраться
3. Не смог двинуться дальше самостоятельно

Важное правило: **Анастасия проверяет preview, не тексты**. То есть, если у тебя вопрос формата «правильно ли я написал этот абзац» — это вопрос к Claude в отдельной сессии, не к Анастасии. Анастасия заходит по твоей ссылке, смотрит на готовый лендинг глазами читателя.

Когда идёшь к Анастасии — формулируй вопрос полностью:

Плохо: «Не работает, помоги»  
Хорошо: «Делал X, Claude Code сказал Y, в preview вижу Z. Ожидал W. Что не так?»

С полным контекстом Анастасия решит за 2 минуты. Без контекста — расспросит 10 минут, а потом решит.
