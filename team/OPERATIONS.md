# OPERATIONS — Процесс работы

Этот документ — про то, как устроена работа команды: от получения задачи до публикации лендинга. Пошаговая инструкция и чек-листы.

Читать, когда ты уже прочитал START_HERE, 01, 02, 03 и получил первую задачу.

---

## Три типа задач — три разных процесса

### Тип А. Редактировать тексты существующего лендинга

Пример: «Перепиши подзаголовок в блоке "Масштаб сдвига"».

**Короткий процесс:**

1. Открой Claude (claude.ai), приложи `docs/landing-copy-final.md` и `team/03-text-tone.md`
2. Обсуди правку, получи финальный текст
3. Открой Claude Code for Web (claude.ai/code)
4. Создай новую сессию в проекте hrasc-lab
5. Промпт: «В ветке [имя твоей ветки, любое] в файле `index.html` найди текст "[старый текст]" в секции "[название секции]" и замени на "[новый текст]". Больше ничего не трогай. Пришли preview-ссылку»
6. Получи ссылку, проверь
7. Отправь Анастасии

### Тип Б. Правка/добавление блока существующего лендинга

Пример: «Добавь FAQ-секцию с 5 вопросами» или «Перенастрой блок цифр на новые данные».

**Процесс средней длины:**

1. Открой Claude, обсуди что именно нужно добавить/изменить: текстовое содержание, формат, место в лендинге
2. Собери итоговый текст
3. Открой Claude Code for Web
4. Промпт с описанием секции (см. шаблоны ниже)
5. Получи preview, проверь
6. Итерации правок, если нужно
7. Финальный preview — отправь Анастасии

### Тип В. Создание нового лендинга

Пример: «Собери новый лендинг на hrasc-lab.ru/[имя-проекта]».

**Длинный процесс, 1-3 рабочих дня:**

1. Обсудить структуру и позиционирование в Claude
2. Написать все тексты в документ `docs/landing-copy-[имя].md`
3. Загрузить документ в репозиторий
4. Передать в Claude Code for Web, собрать лендинг
5. Итерации в preview
6. Финальная проверка
7. Отправить Анастасии

Подробнее — ниже.

## Как создать новый лендинг от начала до конца

### Шаг 1. Договорись с Анастасией о задаче

Прежде чем что-то делать — уточни:

- Какой адрес будет у лендинга (например, `hrasc-lab.ru/research-compression`)
- В чём смысл лендинга (кто читатель, какое действие должен совершить)
- Есть ли дедлайн
- Какие материалы уже есть (исследования, цифры, картинки, существующие тексты)

Запиши всё это в заметку. Это станет основой для работы с Claude.

### Шаг 2. Обсуди структуру в обычном Claude

Открой claude.ai, начни новую сессию.

Приложи в начале:
- `team/how-to-work-with-me.md` (или `team/01-claude-guide.md` + `team/03-text-tone.md`)
- `docs/landing-copy-final.md` (как эталонный пример тона)

Напиши:

```
Я работаю в команде Исследовательской лаборатории hrasc-lab.ru. 
Делаю новый лендинг для [описание проекта].

Читатель — [кто].

Главное действие, которое читатель должен совершить — [что].

Контекст и материалы: [прикрепи или опиши].

Прежде чем писать тексты, помоги мне собрать структуру лендинга. 
Какие блоки нужны, в каком порядке, о чём каждый. Используй как 
опорный образец основной лендинг hrasc-lab.ru (тексты во вложенном 
документе).

Дай 2-3 варианта структуры, разных по подходу.
```

Получи варианты. Выбери или смешай. Если ни один не подходит — объясни, что не так, попроси переделать.

### Шаг 3. Напиши тексты

Когда структура утверждена — идёшь блок за блоком.

Пример запроса для одного блока:

```
Собираем первый блок (Hero) для лендинга [название].

Задача Hero — [что он должен сделать]. Это первое, что видит читатель.

Структура Hero: kicker, заголовок H1, подзаголовок, две кнопки (primary и secondary).

Дай 3 варианта заголовка, разных по подходу. Под каждым — короткое пояснение, что он делает.

Держи тон, который описан в документе tone (прикреплён).
```

Переходи к следующему блоку, когда предыдущий готов.

**Важно:** не пиши весь лендинг одним запросом. Качество резко падает. Блок за блоком.

### Шаг 4. Собери все тексты в единый документ

Создай файл `landing-copy-[имя].md` по шаблону основного лендинга. Структура файла:

```markdown
# [Название проекта] · Тексты лендинга

**Дата:** [дата]
**Для:** Claude Code — сборка лендинга

---

## Сквозные решения

- Цель лендинга: [...]
- Читатель: [...]
- Основное действие: [...]
- Регистр: деловой консалтинговый

---

## Шапка

- Логотип-текст: Исследовательская лаборатория (или другое имя если нужно)
- Пункты меню: [...]
- CTA-кнопка: [...]

---

## Блок 1 · Hero

**Kicker:** [...]
**Заголовок:** [...]
**Подзаголовок:** [...]
**CTA primary:** [...]
**CTA secondary:** [...]
**Визуал:** [что за картинка]

---

## Блок 2 · [название]
[тексты]

[и так далее по всем блокам]

---

## Футер

[содержание футера]
```

Смотри как сделан `docs/landing-copy-final.md` — это эталон. Твой документ должен быть в том же формате.

### Шаг 5. Загрузи документ в репозиторий

1. Открой репозиторий https://github.com/stacyssummers/hrasc-lab
2. Зайди в папку `docs/`
3. Нажми «Add file» → «Upload files»
4. Перетащи свой `landing-copy-[имя].md`
5. Внизу страницы — поле «Commit changes». Напиши краткое описание: `docs: add copy for [имя] landing`
6. Оставь «Commit directly to the main branch»
7. Нажми «Commit changes»

Файл теперь в репозитории, Claude Code его увидит.

### Шаг 6. Собери лендинг в Claude Code for Web

Открой claude.ai/code. В проекте hrasc-lab создай новую сессию.

Промпт (базовый шаблон):

```
Привет. Задача — собрать новый лендинг на основе готового документа 
с текстами.

**Контекст.**
- Репозиторий: hrasc-lab
- Для ориентира обязательно прочитай:
  - docs/landing-copy-[имя].md — тексты нового лендинга
  - docs/design-system.md — дизайн-система
  - index.html существующего лендинга — как образец реализации 
    дизайн-системы

**Что нужно сделать.**
1. Создай новую ветку (называй как угодно на твой выбор, 
   например feat/research-compression)
2. Создай новый файл [путь].html (например, 
   research-compression/index.html или compression.html 
   в корне — на твой выбор того, что удобнее с точки зрения 
   Vercel и cleanUrls)
3. Собери в нём лендинг по документу с текстами
4. Используй дизайн-систему: цвета, шрифты, сетку, типовые блоки
5. Подключи favicon и apple-touch-icon (из файлов, которые 
   уже есть в репозитории)
6. Проверь, что лендинг выглядит в мобильной версии

**Чего НЕ делать.**
- Не трогай main
- Не трогай основной лендинг (index.html в корне) и его демо
- Не мержи в main — просто пришли preview-ссылку

**После работы.**
Пришли:
- Ссылку на созданную ветку
- Ссылку на preview от Vercel
```

Claude Code за 10-30 минут соберёт лендинг. Смотри прогресс, отвечай на уточняющие вопросы, если возникнут.

### Шаг 7. Проверь preview

Получишь ссылку вроде `https://hrasc-lab-git-[ветка].vercel.app/[путь]`. Открой её.

**Чек-лист проверки preview:**

**С десктопа:**
- [ ] Лендинг открывается без ошибок
- [ ] Все блоки на месте в правильном порядке
- [ ] Заголовки, подзаголовки, тексты — такие, как должны быть
- [ ] Кнопки выглядят как кнопки и при клике что-то делают
- [ ] Картинки загрузились
- [ ] Шрифт — Inter (буквы современные, без засечек)
- [ ] Фон — тёплый кремовый `#fafaf8`, не белый
- [ ] Акцентный цвет — бордовый `#5c1c14`, используется точечно

**С мобильного** (открой ту же ссылку с телефона):
- [ ] Лендинг не уходит горизонтально за край экрана
- [ ] Текст читается без зума
- [ ] Все секции помещаются вертикально и идут друг за другом
- [ ] Кнопки нажимаемые (не слишком мелкие)
- [ ] Картинки масштабируются под мобильный размер

**По содержанию:**
- [ ] Нет орфографических ошибок
- [ ] Нет оборванных предложений («как настрое... » где-то)
- [ ] Нет плейсхолдеров типа «[здесь будет текст]» или «Lorem ipsum»
- [ ] Имена, названия, цифры — все правильные
- [ ] Тон выдержан по всем блокам (см. чек-лист в `03-text-tone.md`)

Если что-то не так — пиши Claude Code правки в той же сессии. Он будет знать контекст.

### Шаг 8. Отправь Анастасии

Когда preview проверен и тебя всё устраивает — пишешь Анастасии в Telegram сообщение вида:

```
Лендинг [название] готов к ревью.

Preview: [ссылка]
Ветка: [имя ветки в GitHub]

Что проверить особенно — [если есть сомнения в чём-то конкретном]
```

Анастасия смотрит preview. Дальше два варианта:

**Вариант 1.** Говорит «публикую». Делает merge в main. Через несколько минут лендинг живёт на `hrasc-lab.ru/[путь]`. Всё.

**Вариант 2.** Присылает правки. Ты открываешь свою сессию в Claude Code for Web (или новую, если старая закрылась), вносишь правки, отправляешь новый preview. Цикл повторяется.

## Шаблоны промптов для Claude Code

### Добавление нового блока на существующий лендинг

```
В ветке [имя] в файле [путь].html после секции [название предыдущей] 
добавь новый блок.

Параметры блока:
- Kicker: [...]
- Заголовок: [...]
- Формат: [карточки / таймлайн / три колонки / параграфы]
- Содержание: [...]
- Фон: основной #fafaf8 или вторичный #f7f5f1

Используй дизайн-систему из docs/design-system.md.
Сделай мобильную адаптацию.
```

### Замена картинки в хиро

```
В ветке [имя] в файле [путь].html в секции Hero замени картинку.

Старая картинка: files/pictures/[старое имя]
Новая: files/pictures/[новое имя] (уже загружена в репозиторий)

Картинка должна быть центрирована, max-width 360px на десктопе, 
max-width 260px на мобильных.
```

### Правка формы обратной связи

```
В ветке [имя] в файле [путь].html в модалке контактной формы 
добавь/удали/измени поле.

Что именно: [подробное описание]

Endpoint /api/submit не трогать — это единая инфраструктура 
для всех лендингов.
```

## Что делать, если что-то пошло не так

### Claude Code не отвечает / висит

Иногда стрим прерывается. Решение: напиши в той же сессии «продолжи с места, где остановился». Claude Code помнит контекст.

Если не помогает — закрой вкладку, открой заново, перейди в ту же сессию (она в списке слева). Напиши «продолжи работу по задаче [кратко напомни]».

### Claude Code сделал не то, что ты просил

Случается. Решения по приоритету:

1. Напиши точно что не так и что должно быть: «В блоке X ты сделал Y, а нужно Z. Поправь». Claude Code исправит в следующем коммите.

2. Если правок много и кажется, что легче переделать — попроси откатить: «Откати последний коммит, начнём этот блок с нуля». Он умеет.

3. Если совсем ничего не получается — напиши Анастасии, покажи что происходит. Она разберёт.

### Preview не открывается

Varianты:
- Не успел собраться. Подожди 2-3 минуты после того, как Claude Code написал «готово»
- Закрыта админка GitHub или Vercel. Проверь, что тебе действительно дали доступ
- Опечатка в URL. Скопируй заново из сообщения Claude Code

### Получил ссылку, открыл — совсем другой лендинг

Это значит, что ты открыл preview чужой ветки или preview main-ветки (основной лендинг). Проверь, что в ссылке указана твоя ветка.

### Сделал лендинг, но он криво выглядит

Не паникуй. Открой новую сессию Claude Code, напиши: «В ветке [имя] в файле [путь].html визуально не работает секция X. На скриншоте видно [опиши]. Разберись и исправь».

Прикрепи скриншот, если можешь. Claude Code поймёт.

## Правила хорошего тона в работе

- **Не работай в main.** Всегда в своей ветке. Если случайно попал в main — останови работу, напиши Анастасии
- **Не мержи сам.** Merge делает только Анастасия
- **Не удаляй чужие ветки** на GitHub
- **Не меняй** настройки репозитория, проекта на Vercel, структуру папок без согласования
- **Отправляй preview только когда готово.** Не «в процессе, посмотрите что пока есть» — это тратит Анастасии время на то, что ещё будет меняться
- **Пиши ясные commit-сообщения** в Claude Code (он делает это сам, но если просишь — пусть пишет по делу, не «updates»)
- **Сохраняй важные ссылки** на ветки и preview в заметку. Они не пропадут, но искать потом по переписке долго

## Чек-лист перед отправкой Анастасии

Прежде чем нажать «Отправить» в Telegram — пройди:

### По процессу
- [ ] Работал в своей ветке, не в main
- [ ] Финальный preview — это действительно финальная версия, не «в процессе»
- [ ] Все тексты загружены в `docs/landing-copy-[имя].md` и закоммичены
- [ ] Картинки лежат в `files/pictures/`
- [ ] Ссылка на preview проверена (реально открывается и показывает лендинг)

### По контенту
- [ ] Прошёл чек-лист из `03-text-tone.md`
- [ ] Нет опечаток
- [ ] Нет плейсхолдеров
- [ ] Все ссылки работают (клик по всем кнопкам)
- [ ] Форма обратной связи, если есть — отправляет данные (заполни тестово, проверь)

### По визуалу
- [ ] Прошёл чек-лист из `02-visual-guide.md`
- [ ] Открыл с мобильного, всё работает
- [ ] Открыл с десктопа, всё работает
- [ ] Шрифт Inter, фон кремовый, акцент бордовый

Если все галочки стоят — отправляй. Если хотя бы одной не хватает — доделай сначала.

## После того как Анастасия одобрила

Лендинг опубликован на `hrasc-lab.ru/[путь]`. Твоя работа над этой задачей закончена.

Что остаётся:

- **Не удаляй свою ветку** сразу. Пусть полежит месяц-два на случай, если нужно будет откатить
- **Сохрани ссылку** на финальный лендинг и на `landing-copy-[имя].md` в общей базе знаний команды (формат этой базы — уточни у Анастасии)
- **Следи за аналитикой** первую неделю, если Анастасия попросит. Если нет — можешь переключаться на следующую задачу
