# START HERE

Привет. Ты попал в команду Исследовательской лаборатории — проекта НКО «Люди. Бизнес. Будущее.». Мы делаем лендинги на домене hrasc-lab.ru для разных направлений Лаборатории: основной лендинг, лендинги под исследования, под события, под клиентские проекты.

Этот документ — первое, что ты читаешь. После него ты знаешь, куда идти дальше и что делать.

---

## Что ты будешь делать в команде

Создавать и редактировать лендинги под разные задачи Лаборатории. Технически это значит:

- Писать и править тексты лендингов
- Работать с Claude в двух формах: обычный чат (обсуждение, редактура) и Claude Code for Web (сборка кода сайта)
- Проверять результат на preview-версии
- Отдавать готовый лендинг на ревью Анастасии, которая его публикует

Тебе **не нужно**:

- Уметь программировать
- Знать HTML, CSS, JavaScript
- Работать с терминалом или командной строкой
- Устанавливать программы на компьютер

Всё делается в браузере через GitHub и Claude.

## Чек-лист первых дней

### День 1. Создание аккаунтов

Тебе нужно завести аккаунты в трёх сервисах. Если какие-то уже есть — используй их.

**1. GitHub** (где живёт код сайта)
- Идёшь на https://github.com
- Кнопка «Sign up» наверху справа
- Регистрируешь аккаунт на свою рабочую почту
- Подтверждаешь email

**2. Claude** (ИИ-ассистент для обсуждений и редактуры)
- Идёшь на https://claude.ai
- Кнопка «Sign up» или «Log in»
- Регистрируешь аккаунт на ту же рабочую почту
- Получаешь бесплатный тариф — его хватит для начала, но скоро понадобится платный

**3. Claude Code for Web** (ИИ-помощник для сборки сайта)
- Это отдельный продукт на https://claude.ai/code
- Логинишься тем же аккаунтом, что в Claude
- Подключаешь свой GitHub-аккаунт (Claude Code попросит разрешение при первом использовании)

**4. Vercel** (где сайт «живёт» и показывается в браузере)
- Идёшь на https://vercel.com
- Кнопка «Sign Up»
- Выбираешь «Continue with GitHub» — это привяжет твой GitHub к Vercel
- Подтверждаешь разрешения

Когда все четыре аккаунта готовы — **пиши Анастасии**, чтобы она добавила тебя в команду. Пришли ей:
- Свой GitHub username (имя пользователя на GitHub, вида `@yourname`)
- Email, который ты использовал для регистрации

Без этого шага ты не сможешь работать с кодом сайта — нужны права доступа, которые даёт только Анастасия.

### День 3. Первая реальная задача

Анастасия поставит тебе первую задачу — либо редактирование существующих текстов, либо создание нового лендинга. Дальше работаешь по инструкциям.

## Три типа задач, которые тебе будут ставить

### 1. Редактирование текстов существующего лендинга

«На лендинге поменяй текст такого-то блока» — короткая задача. Работаешь с текстом в обычном Claude (если нужно переформулировать), потом открываешь Claude Code for Web, просишь применить правку в коде, получаешь preview-ссылку, отправляешь Анастасии.

### 2. Создание нового лендинга

«Сделай лендинг для нового проекта» — это большая задача в несколько шагов.

Сначала обсуждаешь структуру и тексты в обычном Claude, собираешь документ `landing-copy-[название].md`. Потом в Claude Code for Web передаёшь промпт для сборки сайта. Получаешь preview-ссылку, правишь если нужно, отправляешь Анастасии.

Подробная инструкция — в `OPERATIONS.md`.

### 3. Добавление/правка блоков существующего лендинга

«Добавь секцию с FAQ» или «перенастрой блок цифр под новые данные». Средняя по объёму задача — открываешь Claude Code for Web, описываешь что нужно, получаешь preview, отправляешь Анастасии.

## Как устроена работа с Анастасией

Ты не работаешь в main — основной ветке репозитория, которая показывается на сайте. Ты работаешь в отдельных ветках (это как черновик), которые Claude Code for Web создаёт автоматически.

**Анастасия проверяет preview, а не тексты.** Это значит:

- Ты сам доводишь тексты до финального состояния
- Ты сам собираешь лендинг на preview
- Анастасия заходит по твоей ссылке, смотрит результат глазами пользователя
- Дальше она либо говорит «публикую» и делает merge в main, либо присылает правки по тому, что увидела

Эта модель работает, только если ты серьёзно относишься к самопроверке. Перед тем как отправлять ссылку, ты должен быть уверен, что лендинг готов — не «почти готов, Анастасия допилит», а готов.

Что это значит на практике:

- Тексты финальные, а не черновики
- Preview проверен с десктопа и с телефона
- Все кнопки и ссылки ведут туда, куда должны
- В тексте нет орфографических ошибок и оборванных предложений

Если ты в чём-то сомневаешься — лучше обсудить со мной (Claude) в отдельной сессии до отправки preview, а не просить Анастасию разбираться в проблемах. Её время — ограниченный ресурс всей команды.

## Что тебе можно, что нельзя

### Можно

- Создавать новые лендинги в репозитории в своих ветках
- Редактировать тексты в документах `docs/landing-copy-*.md`
- Использовать Claude для обсуждений, редактуры, разбора задач
- Использовать Claude Code for Web для сборки кода
- Смотреть preview-версии
- Задавать вопросы Анастасии в любой момент

### Нельзя

- Делать merge в main (публиковать изменения на live-сайт) — это делает только Анастасия
- Менять настройки репозитория на GitHub
- Менять настройки проекта на Vercel
- Удалять файлы из репозитория без согласования
- Трогать папки `api/` (это техническая часть, которую писала Анастасия) без её разрешения

## Границы — что не обсуждается с ИИ

Если ты работаешь над лендингом под клиентский проект — **не пересылай в Claude конфиденциальные документы клиента**. Никакие:

- Переписки с клиентом
- Внутренние документы клиента
- Данные сотрудников клиента
- Результаты оценок

Клиентский контекст обсуждается на голосовых или в защищённых каналах, не в открытых ИИ-чатах. Если сомневаешься, можно ли что-то прислать в Claude — спроси у Анастасии.

Для лендингов это почти никогда не возникнет, но знать надо.
