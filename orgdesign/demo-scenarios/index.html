Сценарий А · Сократить средний управленческий уровень

Между топ-менеджментом и линейными руководителями — три уровня среднего управления. Предложение: убрать средний слой целиком, перевести его задачи частью наверх, частью вниз.

Превышение нормы управляемости в 7 дирекциях

После сокращения среднего слоя у руководителей верхнего уровня в подчинении окажется по 18–24 человека вместо рекомендуемых 7–9. Качество управления падает уже в первый квартал.

Критические роли без замены — 12 позиций

Среди сокращаемых — носители уникальной экспертизы (главные технологи участков, ведущие инженеры по качеству). На рынке такие специалисты не закрываются за 2–3 месяца.

Риск каскадных уходов

Линейные руководители, лишившиеся прямого начальника-эксперта, традиционно начинают искать работу в течение 6–9 месяцев после реорганизации. Прогноз дополнительных уходов: 8–12% линейного руководящего состава.

Цель по деньгам достигнута. Через 6 месяцев структура потребует достройки — частично откатывая экономию.

Сценарий Б · Объединить региональные подразделения

Два региональных дивизиона работают как зеркальные структуры — у каждого свой бэк-офис, своя финансовая служба, свой HR. Предложение: объединить дублирующие функции в единый сервисный центр.

Сокращение затрагивает 96 позиций бэк-офиса.

Норма управляемости — в пределах нормы

Объединение не нарушает управленческую вертикаль. Все руководители сохраняют разумное число подчинённых.

Региональная автономия в принятии решений снижается

Скорость реакции на запросы производства из регионов вырастет с 1–2 дней до 3–5 рабочих дней в первые полгода — пока сервисный центр выстраивает процессы.

Критических ролей без замены — нет

Сокращаемые функции дублированы между регионами, экспертиза сохраняется на стороне объединённого центра.

Структурно решение чистое. По деньгам — закрывает только две трети цели. Нужен либо второй шаг, либо комбинация с другим сценарием.

Сценарий В · Вывести сервисные функции на аутсорс

IT-поддержка, клининг, охрана, часть бухгалтерии — функции, в которых рынок предлагает зрелые внешние сервисы. Предложение: передать эти функции подрядчикам.

Полностью передать ИТ и бухгалтерию подрядчику нельзя — остаются позиции «контролёров контракта», «владельцев процесса», «приёмки работ». Прогноз: 35–50 позиций придётся вернуть в течение года.

Зависимость от подрядчика по критическим процессам

ИТ-поддержка производственных систем — точка отказа. Замена подрядчика занимает 4–6 месяцев. На этот срок производство становится заложником качества внешнего сервиса.

Скрытый рост накладных

В первый год расходы на управление подрядчиками, юридическое сопровождение и контроль качества прибавляют ещё 80–120 млн ₽ к стоимости решения.

На бумаге — лучшая экономия. По полной стоимости решения — близко ко второму сценарию. Главный риск — потеря управляемости критическими процессами.

Ни один из трёх сценариев не закрывает задачу полностью.

Сценарий А — даёт деньги, но создаёт структурный долг, который вернётся откатами через полгода.

Сценарий Б — структурно чистый, но цель по деньгам не достигает.

Сценарий В — лучший по виду, худший по полной стоимости после учёта подрядчиков и оставленных контролёров.

Оптимум — гибрид: объединение бэк-офиса (Б) + точечный аутсорс непроизводственных функций (часть В), без сокращения управленческого слоя. Расчётная экономия — 740 млн ₽/год при структурно сохранной модели.

В реальном проекте таких сценариев — не три, а двадцать. Микросервис позволяет держать их параллельно, считать в моменте и сравнивать по сопоставимым метрикам.
